# 02-Multi-Agentic-AI-RAG-With-Vector-Database

Here’s a **clean, professional GitHub README (paragraph format)** for your **PDF RAG Agentic AI system** 👇

---

This project is an advanced **Agentic AI-powered PDF Assistant** that enables intelligent question-answering over documents using a **Retrieval-Augmented Generation (RAG)** architecture. Built with the **phi framework**, it integrates **Groq’s LLaMA 3.3 model** for fast inference and **Google Gemini embeddings** for semantic understanding. The system ingests PDF documents from URLs, processes them into vector embeddings, and stores them in a **PostgreSQL-based vector database (pgvector)** for efficient retrieval. A specialized AI agent is then configured with access to this knowledge base, allowing it to perform contextual search, retrieve relevant document chunks, and generate accurate, grounded responses. The assistant also supports **persistent storage and session memory** using PostgreSQL, enabling chat history tracking and multi-turn conversations. Designed with modularity and scalability in mind, this solution demonstrates key **Agentic AI concepts** such as tool usage, knowledge retrieval, memory management, and autonomous reasoning, making it highly suitable for enterprise use cases like document intelligence, knowledge assistants, and internal search systems.
